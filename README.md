# 🚨 应急快报

> 多人协作灾害事故信息快报系统

现场人员（约50人）通过华为手机采集文字、照片、语音等现场信息，AI 后台每10分钟自动整合提炼一版结构化快报，支持编辑审阅后一键报送上级单位。

## 产品定位

面向灾害事故现场，解决"信息快报生产不及时"的核心痛点。全华为生态，鸿蒙原生应用 + PC 网页后台。

## 核心功能

| 模块 | 说明 |
|------|------|
| 📱 信息员端（鸿蒙 App） | 拍照、语音、文字一键上报；查看当前快报 |
| 🤖 AI 后台引擎 | 多模态理解（语音转文字、照片OCR）；每10分钟增量生成；矛盾检测标注 |
| 💻 编辑审阅端（PC Web） | 审阅AI初稿；对比变化；编辑定稿；导出 Word/PDF |
| 📲 指挥端（鸿蒙 App） | 只看定稿版；多事件切换；回溯历史版本 |

## 角色说明

| 角色 | 人数 | 终端 |
|------|------|------|
| 现场信息员（Reporter） | ~45人 | 华为手机 App |
| 编辑审核员（Reviewer） | ~3人 | 手机 / PC 网页 |
| 指挥官（Commander） | ~2人 | 华为手机 App |
| 系统管理员（Admin） | 1人 | PC 后台 |

## 10分钟滚动周期

```
00'  信息员持续上报素材
08'  AI 开始整合本周期素材
10'  初稿推送给编辑审阅
↓    编辑审阅（不限时）
✓    定稿后推送指挥官
```

## 技术栈

- **前端（手机）**：鸿蒙 ArkTS + ArkUI
- **前端（PC）**：Vue 3 / React
- **后端**：Node.js / Go
- **AI 引擎**：华为云 AI 服务（语音转文字、OCR、文本生成）
- **部署**：政务云 / 私有化部署
