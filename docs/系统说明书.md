> 版本 0.1.0 | 最后更新：2026-06-21

---

## 一、系统概述

应急快报是一套面向应急指挥场景的**信息采集 → 素材管理 → AI 自动快报生成**一体化平台。支持网页端操作与飞书机器人消息上报，可实现现场文字、图片、视频等多类型素材的实时采集，并通过 AI 引擎自动生成应急快报。

### 核心功能

| 功能 | 说明 |
|------|------|
| **事件管理** | 创建、查看、关闭应急事件 |
| **素材采集** | 支持文字、照片、视频多类型素材上报 |
| **飞书机器人上报** | 通过飞书消息直接上报文字、图片、视频素材 |
| **AI 自动快报** | 基于素材内容自动生成应急快报 |
| **快报审阅** | 编辑、对比、定稿、导出（Word / HTML） |
| **系统管理** | 用户管理、角色权限、邀请码、API 密钥、模板配置 |

### 6.2 支持的消息类型

| 消息类型 | 说明 | 处理结果 |
|----------|------|---------|
| `text` | 文字消息 | 作为文字素材上报当前事件 |
| `image` | 图片消息 | 下载图片 → 作为 photo 素材上报 |
| `file` / `media` | 视频/文件消息 | 下载文件 → 根据扩展名判断 video 或 voice 素材上报 |

### 6.3 飞书指令

| 指令 | 功能 |
|------|------|
| 直接发送文字 | 上报文字素材 |
| 发送图片 | 上报图片素材 |
| 发送视频/文件 | 上报视频素材 |
| `帮助` / `help` / `状态` / `菜单` | 查看帮助 |
| `事件` / `活跃事件` | 查看事件列表 |
| `选择事件 [关键词/ID]` | 选择上报的事件 |
| `当前事件` | 查看当前选中的事件 |
| `取消` / `清除` / `切换事件` | 清除事件选择 |
| `事件详情 [ID]` | 查看事件详情 |
| `今日快报` / `今日简报` | 查看今日定稿快报 |
| `设置名字 [姓名]` | 修改飞书用户的显示名称 |

### 6.4 用户自动绑定

飞书用户首次发送任何消息时，系统自动：
1. 获取用户的 `open_id`
2. 在 `users` 表创建记录（角色: reporter，显示名: `飞书用户_xxxxxx`）
3. 后续所有素材上报以该用户的身份提交
4. 可通过 `设置名字 [姓名]` 自定义显示名

## 九、常见问题

### Q: 飞书消息发出去没有回复？

- 检查 `pm2 logs yingji-backend` 是否有收到消息日志
- 确认飞书机器人应用已在飞书开放平台启用
- 确认 `.env` 中 `FEISHU_APP_ID` 和 `FEISHU_APP_SECRET` 配置正确

### Q: 图片/视频上传后在前端不显示？

- 检查数据库 `materials` 表中 `file_path` 字段是否以 `/` 开头
- 网页端上传存为 `/uploads/photos/xxx.jpg`（有前导 `/`）
- 飞书传入存为 `uploads/feishu-photos/xxx.jpg`（无前导 `/`）
- 前端已做兼容处理，两种格式均可正常显示

### Q: 上传人显示为"测试员"而不是飞书用户？

- 确认飞书用户是否已自动注册（查看 `users` 表 `feishu_open_id` 字段）
- 首次飞书消息会自动注册，再次发送即可使用
- 使用 `设置名字 [姓名]` 修改显示名称

### Q: 如何自定义快报模板？

- 系统管理 → 快报模板卡片 → 编辑模板
- 支持 HTML/JSON 格式配置
